三种可行方案


方案 A：PyInstaller 打包（最简单）


text
text
ai-cockpit.exe    ← Python 后端 + 前端静态文件
├── 启动 FastAPI 服务 (localhost:8080)
├── 自动打开浏览器访问 localhost:8080
└── Camoufox 首次运行自动下载浏览器内核



优点：最快实现，一个 exe 搞定
缺点：Camoufox 浏览器内核需要首次运行时下载，exe 本身约 30-50MB
适合：个人工具、本地使用


方案 B：Electron + Python 子进程（你熟悉的方式）


text
text
AI Cockpit.exe     ← Electron 壳
├── 内嵌 Python 后端（PyInstaller 打包的子进程）
├── Electron 窗口渲染前端
└── 系统托盘 + 开机自启



优点：用户体验最好，你之前 mimo-bot、multi-user-browser 都是这个路线
缺点：打包体积大（Electron ~150MB + Python ~50MB）
适合：分发给其他人用


方案 C：Nuitka 编译（性能最优）


text
text
ai-cockpit.exe    ← Python 编译成原生 C 二进制
├── 启动速度比 PyInstaller 快 2-3 倍
├── 反编译难度高（保护源码）
└── 体积比 PyInstaller 小



优点：启动快、体积小、源码保护好
缺点：编译时间长，兼容性需要测试


我的建议



你已经有 Electron 经验，走方案 B 最顺手：



1.

2.
PyInstaller 打包 Python 后端 → backend.exe
3.
Electron 壳包起来，启动时 spawn backend.exe
4.
Electron 窗口加载 http://localhost:8080



打包后大概 200-250MB（Electron 150MB + Python 50MB），和你之前的 mimo-bot 差不多体量。

## 用户使用流程

``
1. 下载 zip → 解压
2. 双击 ai-cockpit.exe
3. 首次运行 → 自动下载 Camoufox 浏览器内核（~300MB，有进度提示）
4. 之后自动打开浏览器 → 看到 AI Cockpit 界面
5. 后续启动直接秒开



如果想手动触发（不打 tag）



去 Actions 页面 → 左侧选 "Build Windows EXE" → "Run workflow" → 选 main 分支 → 点绿色按钮。




构建成功后在 Actions 页面底部 Artifacts 区域下载 zip。
