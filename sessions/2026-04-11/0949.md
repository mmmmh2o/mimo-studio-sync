### 🤖 AI

已经完整阅读了 mimo-bot 的全部代码。以下是项目全景分析：





🧠 MiMo Bot — 项目全景


一句话总结

用 Playwright 控制真实浏览器，自动化操作 MiMo/ChatGPT/DeepSeek 等 AI 网页，通过可视化流程编排多轮对话。





架构分层


text
text
┌─────────────────────────────────────────────┐
│  Renderer (Vue3 + Element Plus + vue-flow)  │  ← 6 个页面
├─────────────────────────────────────────────┤
│  Preload (IPC Bridge, 40+ 接口)             │  ← 安全隔离层
├─────────────────────────────────────────────┤
│  Main Process (Electron 30)                 │  ← 服务初始化 + IPC 路由
├─────────────────────────────────────────────┤
│  Core Modules (9 个)                        │  ← 业务逻辑
└─────────────────────────────────────────────┘





9 个核心模块


模块	职责	关键设计
engine.js	流程执行引擎	拓扑排序 → 逐节点执行，支持 pause/resume/stop，错误可配置 continueOnError
browser-controller.js	Playwright 浏览器控制	自动找 Chromium（打包/开发/系统三路径），Cookie 持久化，适配器委托
variables.js	变量引擎	`{{var
database.js	SQLite 存储	7 张表（flows/variables/conversations/scraped_data/projects/settings/scrape_tasks），WAL 模式
adapters/	AI 网页适配器	基类定义统一接口，3 个实现：MiMo / ChatGPT / DeepSeek
git-sync.js	GitHub 同步	simple-git，自动 commit + push workspace 目录
scheduler.js	定时调度	node-cron，支持 daily/hourly/weekly/cron 表达式
scraper.js	网页抓取	列表模式（CSS 选择器）或全页模式，自动存库
plugin-manager.js	插件系统	manifest.json + 动态 import + 依赖注入 + 生命周期（init/destroy）




13 种内置节点类型


节点	作用
start	流程入口，初始化输入变量
send-message	发送消息到 AI（模拟打字 + 随机延迟）
wait-reply	等待 AI 回复
extract	从回复中提取变量（正则/代码块）
condition	条件分支（contains/equals/regex/比较）
loop	循环遍历数组
set-variable	设置变量
read-file / save	文件读写
run-command	执行 shell 命令
human-handoff	暂停流程，等待人工接管
delay	延时等待
end	流程终点



节点通过注册表（Map）管理，插件可以注入新节点类型，扩展不需要改核心代码。





3 个 AI 适配器



共同模式：继承 BaseAdapter，实现 sendMessage（模拟人类打字）+ waitForReply（DOM 稳定检测/指示器消失检测）+ extractReply。



MiMo：DOM 稳定检测（连续 3 次内容不变 → 回复完成），选择器兼容多种 class 命名
ChatGPT：检测 stop-button 消失来判断回复完成
DeepSeek：类似 ChatGPT，loading-dots/generating 指示器




前端 6 个页面


页面	路由	功能
Dashboard	/	总览
FlowEditor	/flow	vue-flow 可视化拖拽编排
Variables	/variables	变量管理
Database	/database	SQLite 数据浏览
Settings	/settings	全 UI 配置
Logs	/logs	运行日志


StatusBar 组件（底部状态栏，实时显示运行状态）




安全设计


单实例锁（防多开）
CSP 头（生产环境严格策略）
contextIsolation + nodeIntegration=false
外部链接一律用系统浏览器打开
ASAR 打包 + better-sqlite3 原生模块解包
数据库查询白名单（只允许查 7 张表）




构建/打包


Windows：NSIS 安装包（可选安装目录、桌面快捷方式）
macOS：DMG（hardened runtime + entitlements）
Linux：AppImage + .deb
Playwright Chromium 作为 extraResources 内嵌
GitHub Actions 三平台并行构建，tag 触发自动 Release




整体评价



优点：

架构清晰，模块职责分明，依赖注入做得不错
适配器模式 + 插件系统，扩展性好
安全实践到位（CSP、contextIsolation、单实例）
变量管道过滤器设计精巧（extract_code、extract_json 等）
跨平台打包考虑周全



潜在问题：

浏览器操作依赖 CSS 选择器，AI 网页改版就会挂
DOM 稳定检测有竞态风险（2 秒间隔检测 3 次）
没有测试用例（core/__tests__/ 目录存在但没看到测试文件）
调度器只支持固定频率，不支持"执行完一个再排下一个"
没有错误重试机制（节点失败就直接终止，除非手动配 continueOnError）



整体是个完成度很高的一日项目，代码质量超出预期。
