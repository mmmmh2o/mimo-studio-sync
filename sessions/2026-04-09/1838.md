SPEC.md 介绍如下：





Agent-king 调度平台 — 一句话总结



给一句话想法 → AI 自动拆任务 → 多个 AI 同时写代码 → 自动监控 → 完成交付。





它是什么



一个全自动化 AI 开发工具。你说"做一个 XX"，系统自动把你的想法拆成几十个小任务，派给多个 AI 编码代理并行干活，全程自动监控、自动修 bug、自动推进。




你只管输入那一句话，中间出问题才通知你。





架构（5 个核心模块）


text
text
你的想法 → 拆分器 → 调度器 → 多个Worker(各自tmux) → 事件回传 → 前端面板
              ↓           ↓                            ↓
          tasks.json   循环分配                    WebSocket实时推送



模块	干什么	技术
Task Splitter	把你的想法拆成子任务列表	调一次 LLM API
Scheduler	看谁闲着就派活	循环 5 秒检查
Worker	每个 AI 跑在独立 tmux 里	Claude Code / OpenCode
Monitor	每 30 秒检查：活着没？超时没？上下文爆没？	tmux 轮询
Hooks Receiver	AI 每次操作都报回来	Claude Code 原生 Hooks




数据流（4 个文件，无数据库）


文件	作用
tasks.json	所有任务的状态
events.jsonl	每个 AI 的操作日志（追加写）
progress.json	总进度快照
workers.json	Worker 配置



为什么不用数据库：单用户工具，JSON 文件可读、可编辑、出问题直接看文件。





关键能力



1. 自动拆任务 — 一句话 → LLM 拆成原子任务 → 存到 tasks.json




2. 多 AI 并行 — 默认 3 个 Worker，最大 10 个，各自独立 tmux session




3. 30 秒监控 — 检测 Worker 存活、超时、上下文占用、API 余额




4. 异常自动处理：

超时 >30min → kill + 通知你
上下文 >70% → 自动清空重建
连续失败 3 次 → 跳过 + 标记 error
API Key 耗尽 → 停止所有 + 告警



5. 实时面板 — WebSocket 推送，浏览器看到每个 AI 在干什么





开发分 4 个阶段


阶段	做什么	耗时
Phase 1	核心引擎：拆任务 + 单 Worker 执行 + CLI	3-5 天
Phase 2	稳定性：超时保护 + 断点恢复 + 上下文管理	2-3 天
Phase 3	并行 + Web UI：多 Worker + React 面板	3-5 天
Phase 4	高级：对抗测试 + 知识图谱 + 一键启动	3-5 天




技术栈



Node.js + TypeScript + tmux + WebSocket + React + TailwindCSS




不引入重型框架，不加数据库，不造用户系统。简单直接。
