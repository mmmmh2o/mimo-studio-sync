MiMo PLC Studio 使用指南


1. 打开工具



直接用浏览器打开 plc-generator/index.html，纯前端单文件，不需要服务器，不需要安装任何东西。





2. 基本流程（3步）


text
text
① 填 I/O 点表 → ② 写控制需求 → ③ 点生成




第一步：填 I/O 点表




点「+ 添加 I/O 点」，填入：

地址：X0、Y1、M0 等三菱标准地址
描述：中文名，如"启动按钮"、"电机"
类型：输入/输出/定时器/计数器/寄存器



第二步：写控制需求




用中文描述你想要的控制逻辑，比如：

text
text
按下启动按钮X0，电机Y0运行
按下停止按钮X1，电机停止
运行时指示灯Y1亮




第三步：点「⚡ 生成指令表」




生成后右侧会显示指令表，切换 tab 可以看梯形图或仿真。





3. 快捷方式：用示例



不想手动填？直接点底部的示例按钮：



示例	覆盖内容
启停自锁	最基础的电机启停
定时器	延时启动
计数器	计数到N次触发
顺序启动	多电机依次启动
正反转互锁	双向互锁控制
星三角启动	降压启动
报警系统	SET/RST 置位复位



点击后自动填入 I/O 表 + 描述，直接点生成就行。





4. 两种生成模式



规则引擎（默认，无需配置）

填好 I/O + 描述，直接点生成
覆盖 8 种常见电路模式
不需要网络



AI 模式（更强大）

在头部填入 API URL，如：https://api.openai.com/v1/chat/completions
填入 API Key
选模型（或留空用默认）
指示灯变绿 = 已连接，之后点生成会走 AI
AI 能处理规则引擎覆盖不了的复杂逻辑
AI 失败时自动回退到规则引擎



支持任何 OpenAI 兼容接口：OpenAI、DeepSeek、通义千问、本地部署的模型都行。





5. 输出怎么看



指令表 tab

语法高亮：指令蓝色、设备绿色、数值橙色、注释灰色
点「📋」复制，点「💾」保存为 .il 文件



梯形图 tab

绿色竖线 = 常开触点，红色 = 常闭触点
蓝色圆 = 线圈输出，黄色方框 = SET，橙色 = RST
青色 ↑ = 上升沿，紫色 ↓ = 下降沿



仿真 tab

点击输入的 ON/OFF 指示灯切换状态
实时计算输出结果
用来验证逻辑对不对




6. 实际使用场景举例



场景：要给一台水泵做启停控制



1.
添加 I/O：X0 启动按钮(输入)、X1 停止按钮(输入)、Y0 水泵(输出)
2.
输入：按下X0启动水泵Y0，按下X1停止，需要自锁
3.
点生成 → 拿到指令表 → 复制到 GX Developer 里



场景：复杂逻辑规则引擎搞不定



1.
配置 AI（DeepSeek 也行，便宜）
2.
用自然语言随便描述，AI 会生成正确的指令表

服务商	URL
OpenAI	https://api.openai.com/v1/chat/completions
DeepSeek	https://api.deepseek.com/v1/chat/completions
通义千问	https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
本地 Ollama	http://localhost:11434/v1/chat/completions


③ 模型选择

下拉选模型，或选「自定义」手动输入模型名。留空的话用 API 服务的默认模型。



④ API Key

填入你的密钥。





操作步骤


1.
URL 框填入地址 → 指示灯变 🟢
2.
Key 框填入密钥
3.
点「⚡ 生成指令表」→ 自动走 AI 通道
4.
如果 AI 调用失败，自动回退到规则引擎，不用手动切
